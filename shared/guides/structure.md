# 项目笔记结构

## /projects/[项目名]/

```
projects/
├── project-alpha/
│   ├── README.md              # 项目概述和技术栈
│   ├── api-spec.md            # API 接口规范
│   ├── conventions.md         # 项目编码规范
│   └── notes/
│       ├── 2024-01-01-功能开发.md
│       └── 2024-01-15-问题修复.md
└── project-beta/
```

## 笔记模板

```markdown
# [日期] [模块名] - [功能/问题描述]

## 背景
[描述上下文]

## 做了什么
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 遇到的问题
- [问题1]：[原因] → [解决方案]
- [问题2]：[原因] → [解决方案]

## 学到的经验
- [经验1]
- [经验2]

## 参考资料
- [链接1]
- [链接2]
```

## 使用建议

1. 每天开发结束后花 5 分钟记录
2. 遇到问题及时记录解决方案
3. 定期回顾总结，提取通用经验
4. 将通用经验迁移到 prompts/templates
